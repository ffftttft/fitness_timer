# 健身计时器（Fitness Timer）

Fitness Timer 是一个专注的、接近商业级质量的间歇训练计时器应用，使用 Flutter 构建。  
适用于 HIIT / EMOM / Tabata 等高强度间歇训练，界面风格简洁专业，默认深色主题。

## 功能特性

- **完整训练流**：准备 → 训练 → 休息 → 按轮次循环 → 训练完成。
- **参数持久化**：自动保存上一次使用的训练参数，重新打开应用会自动恢复。
- **专业深色 UI**：
  - 巨大等宽数字显示剩余时间（避免数字跳动抖动）。
  - 不同阶段使用不同主色（准备黄 / 训练橙 / 休息蓝）。
  - 数字外圈有平滑的圆环进度显示。
- **稳定性**：
  - 训练计时页面在运行期间保持屏幕常亮。
  - 阶段结束时会通过本地通知提醒，即使应用在后台或手机锁屏。
- **反馈系统**：
  - 每个阶段最后 3 秒有短促滴声倒计时提示。
  - 阶段切换及训练结束时会触发触觉反馈（震动）。

## 配置模型（WorkoutConfig）

训练由四个核心参数配置：

- **Warm-up（准备，秒）**：正式开始训练前的准备时间（可设为 0）。
- **Work（训练，秒）**：每轮“训练阶段”的持续时长。
- **Rest（休息，秒）**：每轮训练结束后的休息时长（可设为 0）。
- **Rounds（循环轮数）**：完成多少轮 “Work → Rest”。

这些参数通过 `WorkoutConfig` 定义，并由 `shared_preferences` 持久化到本地。

## 使用说明

### 1. 打开应用

- 启动后会进入 **首页（Home）**，上方展示“上次配置”的概要信息：
  - Warm-up / Work / Rest / Rounds 当前数值。

### 2. 调整训练参数（Settings）

1. 在首页点击 **“Adjust parameters”** 按钮，或点击 AppBar 右侧的 **齿轮图标（Settings）** 进入设置页。
2. 在 **Settings** 页面：
   - 使用滑块分别设置：
     - **Warm-up (seconds)**：准备时间。
     - **Work (seconds)**：训练时间。
     - **Rest (seconds)**：休息时间。
   - 使用中间的步进器设置：
     - **Number of rounds**：训练轮数。
   - 下方有一段文字说明当前的声音与触觉反馈策略。
3. 点击右上角 **“Save”** 保存参数并返回首页，配置会自动写入本地，供下次启动使用。

### 3. 选择应用语言

1. 在 **Settings** 页面底部找到 **Language** 区块。
2. 在 Language 中可以选择：
   - **English**：英文界面。
   - **Chinese (Simplified)**：简体中文界面。
3. 选择后界面文案会立即切换，语言设置也会被持久化，下一次启动应用仍保持上次选择。

### 4. 开始训练

1. 返回 **首页**，确认当前配置无误后，点击 **“Start workout”**。
2. 会进入 **Timer** 页面：
   - 顶部显示当前轮次（如：`第 2 / 8 轮` 或 `Round 2 / 8`）。
   - 中间是一个带颜色的圆环进度条，表示当前阶段的进度。
   - 圆环中间是巨大等宽数字的倒计时（MM:SS）。
   - 下方有控制按钮：Start/Pause、Reset、Skip。

### 5. 计时控制

- **Start / Pause**：
  - 点击主按钮可以开始或暂停当前阶段。
- **Reset**：
  - 将训练流重置回初始状态（从准备阶段或第一轮开始）。
- **Skip**：
  - 直接跳过当前阶段，进入下一个合理阶段：
    - 例如从 Work 直接跳到 Rest，或从最后一轮直接进入完成状态。

### 6. 后台行为

当计时在 **Timer** 页面运行时：

- 应用会通过 `wakelock_plus` 阻止设备自动锁屏（保持屏幕常亮）。
- 每个阶段开始时，会为该阶段的结束时间安排一个本地通知：
  - 即使用户切出应用或锁屏，到达阶段结束时间后仍会有系统通知提醒。

### 7. 训练完成

- 当所有轮次完成后：
  - 阶段状态变为 **Done / 完成**。
  - 播放结束提示音。
  - 触发较强的触觉反馈。
  - 计时自动停止并取消后续通知。

## 项目结构（高层）

- `core/`：主题、颜色、通知、持久化、本地化和通用工具。
- `models/`：数据模型，如 `WorkoutConfig`。
- `providers/`：状态管理（`WorkoutConfigProvider`、`TimerProvider`、`AppLanguageProvider`）。
- `views/`：页面视图（`HomePage`、`SettingsPage`、`TimerPage`）。
- `widgets/`：可复用组件（大号时间文本、圆环进度条、控制按钮、确认对话框等）。

## 运行项目

1. 确保本地已安装 Flutter（版本需满足 `pubspec.yaml` 中的 SDK 约束）。
2. 安装依赖：

```bash
flutter pub get
```

3. 在目标平台上运行（以移动端为例）：

```bash
flutter run
```

本仓库可以作为一个「健身计时器」类应用的基础模板，你可以在此基础上继续扩展更复杂的训练计划管理、动作库、统计分析等功能。
